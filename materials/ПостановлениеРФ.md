# Требования к доступности сайтов для инвалидов по зрению

## Постановление Правительства РФ № 102

---

## Цель документа
Установление требований к обеспечению доступности информации о деятельности государственных органов и органов местного самоуправления для инвалидов по зрению на официальных сайтах в интернете.

---

## Основные требования к официальным сайтам

### 1. Управление с клавиатуры
- Информация должна быть доступна при помощи клавиатуры без ограничений по времени нажатия клавиш
- Доступность для вспомогательных технологий (программы экранного доступа)
- Структура и последовательность информации должны считываться вспомогательными технологиями

**Критерии:**
- Информация, структура и смысловые связи определяются программно или доступны в текстовом формате
- Порядок представления информации должен соответствовать смысловой логике контента
- Последовательное ориентирование по контенту должно сохранять значение и возможность управления

### 2. Масштабирование текста
- Текстовая информация должна масштабироваться не менее чем на **200%** без потери функциональности и появления горизонтальной полосы прокрутки

### 3. Доступность документов
- Документы должны быть доступны для чтения вспомогательными технологиями
- Формат PDF должен соответствовать **ГОСТ Р 70176-2022**
- Альтернативные версии документов должны быть доступны для программ экранного доступа

### 4. Нетекстовый контент

**Общие требования:**
- Нетекстовый контент должен иметь текстовую альтернативу, позволяющую преобразовать его в:
  - Увеличенный шрифт
  - Рельефно-точечный шрифт Брайля
  - Речь
  - Специальные знаки
  - Упрощенный язык

**Исключения (требуется только краткое описание):**
- Элементы управления/поля ввода — должны иметь название, описывающее назначение
- Видео/аудио ограниченные по времени
- Тесты или упражнения, теряющие функциональность в текстовом виде
- Контент для получения специфического сенсорного опыта

**Декоративный контент:**
- Если контент используется только для оформления или невидим пользователю — вспомогательные технологии должны его игнорировать

**CAPTCHA (подтверждение человек/машина):**
- Должна быть представлена на государственном языке РФ в доступной форме
- Предоставлять альтернативные формы для различных типов сенсорного восприятия

### 5. Заголовки страниц
- Заголовки должны описывать тему или цель страницы

### 6. Ссылки
- Цель каждой ссылки должна определяться из текста ссылки или текста вместе с контекстом

### 7. Контрастность
- Минимальный коэффициент контрастности: **4,5:1**
- Для укрупненного текста: **3:1**

**Исключения:**
- Текст как часть неактивных компонентов интерфейса
- Декоративный текст
- Текст в логотипах или названиях торговых марок

### 8. Цветовая кодировка
- Интерфейс для передачи данных и обратной связи **не должен использовать только визуальные характеристики** (цвет, форму, размер, местоположение)
- Необходимо использование текста, доступного для вспомогательных технологий, или звука

### 9. Движущийся контент
- Пользователь должен иметь возможность:
  - Приостановить
  - Остановить
  - Скрыть
  - Изменить частоту обновления
- **Запрещены мигающие элементы** (более 3 вспышек в секунду)
- Анимация загрузки допускается только если необходима для понимания процесса

### 10. Элементы интерфейса
- При перемещении указателя на элементы интерфейса не должно изменяться контекст

### 11. Формы
- Все поля форм должны иметь текстовые описания
- При ошибочном заполнении — текстовое сообщение с пояснением ошибки
- Сообщения об ошибках должны быть доступны программам экранного доступа
- Метки или инструкции должны предоставляться при вводе информации

### 12. Шрифты
- Текстовая информация должна представляться с использованием читаемого шрифта
- Шрифт без декоративных элементов
- Обеспечение достаточной контрастности и различимости символов
